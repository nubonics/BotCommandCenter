# QoL Roadmap

A practical rollout order for the next BotCommandCenter quality-of-life features.

## Phase 1, shipping now
- Action Center
  - account hygiene checks
  - quick links to fix common gaps
  - system-health summary

## Phase 2
- Tags / groups
  - tag accounts by farm, proxy pool, risk, or stage
  - filter accounts by tag
  - use tags later for bulk actions and cost allocation

## Phase 3
- Bulk actions
  - bulk edit status, tags, notes, proxies, and banned flag
  - bulk attach shared metadata to many accounts at once

## Phase 4
- Account P&L dashboard
  - expense rollups per account
  - profit and net once a per-account revenue source is defined
  - compare account health and operating cost side by side

## Phase 5
- Global cost allocation modes
  - split evenly
  - split by active accounts only
  - split by selected tags or groups
  - weighted allocation options later if needed

## Phase 6
- Better wall overlays and account health cards
  - tile overlays for account label, status, proxy group, and plan state
  - health indicators for last seen, spreader state, watchdog signals, and wall detection

## Notes
- The P&L dashboard is the only feature that needs one product decision first: what counts as account revenue.
- Everything else can be built directly on the current app structure.
